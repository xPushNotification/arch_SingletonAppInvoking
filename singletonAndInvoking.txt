
    Стандартная проблема приложения вроде писанных на JavaFX / Spring
    в том - что методы вызываются в них индиректно
    то есть они вызываются не программою - а каким то объектом
    который мы даже не видим,
    как же получить доступ к программе запущенной
    так как тотже Spring / JavaFX вообще будут жить в отдельных потоках?

    просто использовать singleton
    от этого инвоки станут такой же частью приложения как если бы
    вызывались непосредственно нами самими (причем вообще без какой либо разницы)
    нужна по сути лишь точка связи с самим приложением

    и это буквально концепция бинов
    которые помимо описанного еще и привязываются к сессиям
    а место где хранятся подобные "синглетоны" называется "контекст"

    -----------------------------------------------------------------------------
    class Application                       <- со стороны приложения управляемого нами
    {
        static Application app;
        Application()
        {
            if(app == null)
            {
                ..создаем данные приложения
            }
            else
            {
                return app;
            }
        }
    }
    -----------------------------------------------------------------------------
    class SomeIndirectCalled                <- со стороны инвокуемых частей
    {
        @SomePath
        void method()
        {
            Application app = new Application();    <- данные связаны - теперь вызов получил доступ к приложению
        }
    }
